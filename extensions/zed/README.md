# Zed extension for Zero

Syntax highlighting and basic editor support for the Zero programming language
in the [Zed editor](https://zed.dev).

This is a development-stage extension. It is installed as a Zed "dev extension"
from this directory; it is not yet published to the Zed extension registry.

## Install

```sh
pnpm run extension:install:zed
```

The script substitutes the absolute path to the vendored tree-sitter grammar
into `extension.toml`, then prints the directory to select in Zed.

Inside Zed:

1. Open the command palette (`ctrl-shift-p` / `cmd-shift-p`).
2. Run **"zed: install dev extension"**.
3. Select `extensions/zed/` (the directory printed by the script).

Open any `.0` file to see highlighting. Reload extensions with **"zed: reload
extensions"** after editing `highlights.scm`, `config.toml`, or other query
files. Edits to `grammar.js` require regenerating the parser:

```sh
cd extensions/zed/tree-sitter-zero
tree-sitter generate
```

When you are done, restore the manifest so the placeholder stays portable:

```sh
git checkout -- extensions/zed/extension.toml
```

## Layout

```
extensions/zed/
├── extension.toml             # Zed extension manifest
├── languages/zero/
│   ├── config.toml            # language id, file types, comments, brackets
│   ├── highlights.scm         # syntax highlight queries
│   ├── brackets.scm           # bracket pair queries
│   ├── indents.scm            # auto-indent queries
│   └── outline.scm            # symbol outline queries
├── scripts/install.mjs        # dev-install helper
└── tree-sitter-zero/          # vendored tree-sitter grammar
    ├── grammar.js
    ├── src/parser.c           # generated by `tree-sitter generate`
    ├── src/scanner.c
    └── ...
```

## Coverage

The vendored grammar parses 100% of the `.0` files in `examples/` and
`conformance/native/pass/` at the time of vendoring. Run the coverage check
yourself:

```sh
cd extensions/zed/tree-sitter-zero
ZERO_REPO=../../.. node scripts/check-against-upstream.mjs
```

## Publishing

Before publishing to the Zed extension registry the vendored grammar should
move to its own repository (e.g. `vercel-labs/tree-sitter-zero`) so the
`[grammars.zero]` block in `extension.toml` can point at a public git URL with
a pinned `rev`. See the upstream Zed guide:

<https://zed.dev/docs/extensions/languages>
